What is Velocitoro?

Velocitoro is a Maven 2.0 plugin. It creates web site documentation for your project. #set( $a = $groovy.invoke("test.groovy","aaa") ) $a

Note that this plugin does not replace the standard 'site' plugin. It has different aim. While 'site' creates technical documentation, velocitoro is aiming to create "marketing" documentation.

Usage in short: Create the web site in src/web. executing mvn com.verhas.maven:maven-velocitoro-plugin:velocitoro will copy all files (except those that have extension .vmi or .vm to target/web. After this velocitoro will process all files that have a "shadow" file with the extension .vm using Apache Velocity and overwrite the original copied content.

What is a shadow file?

Note that if there is a file index.html.vm and index.html then the latter will be processed. The shadow file index.html.vm may have any content: it is ignored. It's sole purpose is to indicate that index.html has to be processed.

Why not processing index.html.vm? Because this way your IDE editor can rely on the extension of the file and guess the type of the content file when you edit. If your html is not totally mixed up with Velocity macros you may still get a kind of draft view in a browser without executing velocitoro.

.vmi files?

What are .vmi files? The are totally ignored except you can #parse them from any file that is processed using velocity.
